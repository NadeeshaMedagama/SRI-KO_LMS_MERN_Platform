The "Failed to update profile" error has been resolved. Users can now successfully update their profile information including location, phone, and other fields.

‚ùå The Problem

Root Cause: Overly Strict Validation Rules

The profile update was failing due to strict validation rules in the backend middleware:

  • Phone Validation: isMobilePhone() was too strict and rejected valid phone numbers
  • Website Validation: isURL() required full URLs with protocols
  • Social Links: Same URL validation issues
  • Missing Avatar Field: No validation rule for avatar field
// OLD VALIDATION (Too Strict) body('phone') .optional() .trim() .isMobilePhone() // ‚ùå Too strict .withMessage('Please provide a valid phone number'), body('website') .optional() .trim() .isURL() // ‚ùå Requires full URL with protocol .withMessage('Please provide a valid website URL'),

‚úÖ The Solution

Updated Validation Rules

Replaced strict validation with more flexible length-based validation:

// NEW VALIDATION (Flexible) body('phone') .optional() .trim() .isLength({ max: 20 }) // ‚úÖ Just check length .withMessage('Phone number cannot be more than 20 characters'), body('website') .optional() .trim() .isLength({ max: 200 }) // ‚úÖ Just check length .withMessage('Website URL cannot be more than 200 characters'), body('avatar') .optional() .trim() .isLength({ max: 500 }) // ‚úÖ Added avatar validation .withMessage('Avatar URL cannot be more than 500 characters'),

Updated Validation Rules:

  • Name: 2-50 characters (unchanged)
  • Bio: Max 500 characters (unchanged)
  • Phone: Max 20 characters (flexible)
  • Location: Max 100 characters (unchanged)
  • Website: Max 200 characters (flexible)
  • Social Links: Max 200 characters each (flexible)
  • Avatar: Max 500 characters (new)
